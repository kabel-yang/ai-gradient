# ai-gradient
策略梯度算法

策略梯度（Policy Gradient）是强化学习中的一种直接优化策略的方法。它不像DQN那样先学Q值再选动作，而是直接学习一个“动作概率分布”，通过梯度上升让智能体更倾向于选择高回报的动作。

一、核心思想：直接优化策略

• 策略（Policy）：智能体在状态 s 下选择动作 a 的概率分布，记为 \pi(a|s)。

• 目标：最大化期望累积回报 J(\theta)。

• 手段：计算目标函数对策略参数 \theta 的梯度 \nabla_\theta J(\theta)，然后沿梯度方向更新参数：

  \theta \leftarrow \theta + \alpha \nabla_\theta J(\theta)

二、核心算法：REINFORCE

最基础的策略梯度算法，利用整条轨迹的回报进行更新。

1. 算法流程

1.  采样：用当前策略 \pi_\theta 与环境交互，收集一条轨迹 (s_0, a_0, r_1, s_1, a_1, r_2, \dots)。
2.  计算回报：从每个时间步 t 开始，计算累积折扣回报 G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}。
3.  更新策略：对轨迹中的每一步，执行梯度上升：
    \theta \leftarrow \theta + \alpha \gamma^t G_t \nabla_\theta \log \pi_\theta(a_t|s_t)

2. 直观理解

• \nabla_\theta \log \pi_\theta(a_t|s_t)：增加动作 a_t 被选中的概率。

• G_t：权重。如果回报 G_t 很大（动作效果好），则大幅增加该动作的概率；如果回报为负（效果差），则降低概率。

三、进阶技巧与优化

基础REINFORCE方差大、收敛慢，常用以下技术改进：

技巧 作用 说明

基线（Baseline） 降低方差 用 G_t - b(s_t) 代替 G_t，减去一个基准值（如状态价值函数），减少波动。

Actor-Critic 结合值函数 用Critic（价值网络）估计优势函数 A(s,a) 代替 G_t，实现单步更新。

熵正则化 鼓励探索 在损失函数中加入策略的熵，防止智能体过早陷入局部最优。

四、实际运用步骤

1.  定义策略网络：输入状态 s，输出动作概率分布（如通过Softmax）。
2.  交互采样：在环境中运行策略，收集经验数据。
3.  计算损失：计算带基线或优势函数的策略梯度损失。
4.  反向传播：更新策略网络参数 \theta。

五、优缺点对比

• 优点：

  • 直接优化：适用于高维或连续动作空间（如机器人控制）。

  • 收敛性好：理论上能收敛到局部最优。

• 缺点：

  • 高方差：采样估计的回报波动大，需要大量数据。

  • 样本效率低：通常是同策略（on-policy），旧数据不能复用。

策略梯度是解决复杂控制问题（如AlphaGo、自动驾驶）的基石，后续的PPO、SAC等算法都是在它的基础上进行优化。
