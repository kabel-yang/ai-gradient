# ai-gradient
策略梯度算法

策略梯度（Policy Gradient）是强化学习中的一种直接优化策略的方法。它不像DQN那样先学Q值再选动作，而是直接学习一个“动作概率分布”，通过梯度上升让智能体更倾向于选择高回报的动作。

一、核心思想：直接优化策略

• 策略（Policy）：智能体在状态 s 下选择动作 a 的概率分布，记为 \pi(a|s)。

• 目标：最大化期望累积回报 J(\theta)。

• 手段：计算目标函数对策略参数 \theta 的梯度 \nabla_\theta J(\theta)，然后沿梯度方向更新参数：

  \theta \leftarrow \theta + \alpha \nabla_\theta J(\theta)

二、核心算法：REINFORCE

最基础的策略梯度算法，利用整条轨迹的回报进行更新。

1. 算法流程

1.  采样：用当前策略 \pi_\theta 与环境交互，收集一条轨迹 (s_0, a_0, r_1, s_1, a_1, r_2, \dots)。
2.  计算回报：从每个时间步 t 开始，计算累积折扣回报 G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}。
3.  更新策略：对轨迹中的每一步，执行梯度上升：
    \theta \leftarrow \theta + \alpha \gamma^t G_t \nabla_\theta \log \pi_\theta(a_t|s_t)

2. 直观理解

• \nabla_\theta \log \pi_\theta(a_t|s_t)：增加动作 a_t 被选中的概率。

• G_t：权重。如果回报 G_t 很大（动作效果好），则大幅增加该动作的概率；如果回报为负（效果差），则降低概率。

三、进阶技巧与优化

基础REINFORCE方差大、收敛慢，常用以下技术改进：

技巧 作用 说明

基线（Baseline） 降低方差 用 G_t - b(s_t) 代替 G_t，减去一个基准值（如状态价值函数），减少波动。

Actor-Critic 结合值函数 用Critic（价值网络）估计优势函数 A(s,a) 代替 G_t，实现单步更新。

熵正则化 鼓励探索 在损失函数中加入策略的熵，防止智能体过早陷入局部最优。

四、实际运用步骤

1.  定义策略网络：输入状态 s，输出动作概率分布（如通过Softmax）。
2.  交互采样：在环境中运行策略，收集经验数据。
3.  计算损失：计算带基线或优势函数的策略梯度损失。
4.  反向传播：更新策略网络参数 \theta。

五、优缺点对比

• 优点：

  • 直接优化：适用于高维或连续动作空间（如机器人控制）。

  • 收敛性好：理论上能收敛到局部最优。

• 缺点：

  • 高方差：采样估计的回报波动大，需要大量数据。

  • 样本效率低：通常是同策略（on-policy），旧数据不能复用。

策略梯度是解决复杂控制问题（如AlphaGo、自动驾驶）的基石，后续的PPO、SAC等算法都是在它的基础上进行优化。

策略梯度（Policy Gradient）在量化交易中，本质是让 AI 从“预测涨跌”转变为“学习交易行为”。它通过直接优化“买、卖、持有”的动作概率，让模型在长期博弈中最大化收益，而非追求单次预测准确率。

一、核心应用逻辑：从预测到决策

传统量化模型（如 LSTM）是回归/分类问题（预测价格或方向），而策略梯度是决策问题。其优势在于：

端到端优化：直接优化夏普比率、年化收益等交易目标，而非最小化预测误差。

适应复杂规则：天然处理仓位管理、连续交易（如加仓/减仓）等离散动作。

抗噪声：不依赖精确的价格预测，而是学习“在什么状态下该做什么动作”的映射。

二、量化场景的关键设计要素
1. 状态空间（State）

输入特征需包含市场状态，而不仅是历史价格：

市场特征：过去 N 期的 OHLCV（开高低收成交量）、波动率、均线斜率。

持仓状态：当前仓位比例、持仓成本、浮动盈亏（关键，避免模型无视持仓乱下单）。

宏观/情绪：资金流向、板块轮动强度。

2. 动作空间（Action）

根据交易频率设计离散动作（推荐）或连续动作：

离散动作（常用）：

0: 卖出（或平仓）

1: 持有

2: 买入

可扩展为 {-1, 0, 1}表示做空、空仓、做多。

连续动作（进阶）：输出 [-1, 1]的连续值，表示仓位比例（需用 Actor-Critic 结构）。

3. 奖励函数（Reward）

这是策略梯度的灵魂，直接决定模型收敛方向：

基础奖励：r_t = (portfolio_value_t - portfolio_value_{t-1})，即资产净值变化。

风险调整奖励（推荐）：r_t = 收益 - λ * 波动率，或直接使用夏普比率的近似值。

惩罚项：加入交易成本惩罚（如 -0.001 * |position_change|），防止过度交易。

三、实战算法选型与流程
 算法         适用场景      特点

REINFORCE​      入门         低频         简单，但方差大，收敛慢

Actor-Critic​   主流选择    引入价值网络（Critic）评估动作，降低方差

PPO​            工业级推荐​   带裁剪机制，训练稳定，抗过拟合

训练流程（以 PPO 为例）

初始化：策略网络（Actor）和价值网络（Critic）。

交互采样：

用当前策略在历史数据或模拟器中跑一个 Episode（如 1000 个 bar）。

记录每一步的 (state, action, reward, next_state, done)。

计算优势函数：

用 Critic 网络估计状态价值，计算优势函数 A
t=Q(s,a)−V(s)。

这一步是为了告诉模型“这个动作比平均水平好多少”。

PPO 损失更新：

计算策略损失（带裁剪），确保更新步长不会过大破坏策略。

更新 Critic 网络（MSE 损失，拟合真实回报）。

循环：重复直到收益曲线收敛。

四、关键陷阱与风控建议

过拟合（Overfitting）：强化学习极易过拟合到某一段牛市行情。必须进行滚动回测（Walk-Forward Analysis），并严格划分训练/验证集。

市场环境突变：单一模型无法适应牛熊切换。建议引入市场状态编码（如波动率 regime）作为状态输入。

幸存者偏差：回测时务必包含退市股、停牌股等完整数据，避免只选优质股回测。

六、适用边界

适合：中低频趋势跟踪、组合调仓、做市商报价。

不适合：高频交易（延迟要求高）、极度依赖微观结构博弈的策略。

策略梯度在量化中的核心价值在于适应性与自动化。它不追求“猜对明天”，而是通过长期博弈学习一套“赚钱的习惯”。建议先从简单的 REINFORCE + 股票日线数据开始实验，验证奖励函数的有效性。
